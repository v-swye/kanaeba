github小白笔记

Posted on 2020-05-18 | In 编程与技术

开始：基本操作

启动git服务

首先git init在当前目录启动git，读取隐藏的.git文件夹。
一个本地仓库可以接多个远程仓库，代码为：
git remote add [remotename] git@github.com:[username/repository.git]
查看当前目录下的远程连接：git remote
删除远程连接：git remote remove [remotename]

添加跟踪文件，commit与push

在commit前，需要先添加要track的文件到暂存区：git add .
查看状态：git status，绿色表示和暂存区track的文件一致，红色则表示存在未track的文件。
然后commit: git commit -m “comments”
修改commit内容：git commit --amend
在多人合作时，应commit后先pull更新本地到最新版本。
最后push到对应branch，一般为master：git push -u [remotename] master。如果要覆盖已有的远程仓库，将参数调整为-f即可（尽量不要使用）。

回退commit

查看commit记录：git log
回退：git reset --soft HEAD~1，1表示回退到上一步，2表示上上一步，依次类推。注意不要使用--hard，否则会修改覆盖本地仓库的文件。

拉取远程仓库至本地：fetch, merge与pull

pull=fetch+merge，所以安全起见应该先使用fetch，检查差别后再merge。

关于git fetch更专业的表达为git fetch remotename 远程分支:本地分支

退出git

输入:q。

文件查看与删除库

查看本地仓库中的文件：ls -a
删除本地仓库（不删除文件，相当于只是取消git）：rm -rf .git
删除本地文件：rm -rf filepath

分支操作

查看本地所有分支：git branch
新建本地分支：git branch newbranch
将本地分支提交到远程并在远程新建对应分支：

查看本地+远程分支：git branch -a
切换远程分支：git checkout -b [remotename] [remotename/branchname]
切换本地分支：git checkout branchname
删除本地分支：git branch -d [BranchName]
删除远程分支：git push origin --delete branchname
合并分支：首先切换到要保留的主分支branch1，然后git merge branch2。
